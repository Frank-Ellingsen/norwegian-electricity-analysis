# Contributing Guidelines

These guidelines define strict rules for contributing to this repository. All contributors must follow the standards described in `skill.md`.

## 1. Repository Structure
/src
/data
/analysis
/visuals
/docs

Code

## 2. File Rules
- Do not overwrite existing files without explicit approval.
- New files must be placed in the correct directory.
- All file content must be complete and reproducible.

## 3. Coding Standards
- Python: PEP8, modular functions, docstrings, logging.
- SQL: CTEs, explicit columns, ISO dates.
- Data: CSV default, SQLite analytical store.

## 4. Commit Rules
Allowed types: feat, fix, docs, refactor, data, analysis, visuals, chore.

## 5. Branching Rules
feature/<name>
fix/<name>
analysis/<name>
docs/<name>

Code

## 6. Pull Request Rules
Use the required PR template.

## 7. Security Rules
No secrets, credentials, or machine paths.

## 8. Licensing Rules
Do not add or modify license files unless instructed
